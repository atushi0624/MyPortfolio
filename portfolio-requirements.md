## 5. UI/UX機能仕様

### 5.1 ナビゲーション

- 固定ヘッダー（`position: fixed`）でスクロール追従
- PC時: 横並びメニュー（トップ / 実績 / About / 進行フロー / お問い合わせ）
- SP時: ハンバーガーメニュー（タップでスライドイン）
- スクロール時にヘッダー背景に影（`box-shadow`）を付与

### 5.2 アニメーション

- スクロールフェードイン: `Intersection Observer API`で各セクションを表示時に`fade-in` + `translateY`アニメーション
- ホバーエフェクト: カード・ボタンにCSS `transition`の影・拡大効果
- Heroセクション: テキストのフェードイン（初回ロード時）

### 5.3 Heroセクション詳細

- 背景: Veo 3動画（`Analog_to_Digital_Office_Transformation.mp4`）を無音ループ再生
- 動画上に半透明オーバーレイ（`rgba(27,42,74,0.6)`）でテキスト可読性確保
- SP時は動画の代わりにフォールバック静止画（`poster`属性）を表示（パフォーマンス配慮）
- メインコピー・サブコピーは白抜き文字で中央配置

### 5.4 フォーム機能

- Googleフォームとの連携（iframe埋め込みまたはJSによるPOST送信）
- 入力項目: 会社名（必須）、お名前（必須）、メールアドレス（必須）、現在最も課題に感じている業務（任意）
- バリデーション: 必須項目の入力チェック、メール形式チェック
- ボタンテキスト: 「無料診断（オンライン60分）に申し込む」

## 9. パフォーマンス要件

- Hero動画: Web用に圧縮済みであることを確認（推奨: 5MB以下、H.264 / VP9）
- 画像: WebP形式推奨、遅延読み込み（`loading="lazy"`）適用
- CSS / JS: ミニファイ済みで納品
- Lighthouseスコア目標: Performance 80以上（動画背景のためLCPは要調整）
- Core Web Vitals: CLS < 0.1 / INP < 200ms を目標

## 10. Claude Code実装時の注意事項

### 10.1 実装優先度

- **フェーズ1**: 共通パーツ（ヘッダー・フッター・CSS変数・リセットCSS）+ index.html
- **フェーズ2**: 事例詳細ページ×7（共通テンプレート構造で統一）
- **フェーズ3**: アニメーション・レスポンシブ対応・パフォーマンス最適化

### 10.2 コーディング規約

- CSSカスタムプロパティでカラーパレットを一元管理（`--color-primary` 等）
- BEM命名規則を推奨（`.section__title--accent` 等）
- セマンティックHTML（`section` / `article` / `nav` / `main` / `footer`）
- アクセシビリティ: `alt`属性必須、`aria-label`適切に付与

### 10.3 素材プレースホルダー

動画・画像素材が未配置の状態でも動作するよう、プレースホルダー（グレー背景 + テキスト）を設定すること。
